## הודעות פתיחה (העתק לפי המקרה)

### תרחיש 1 — התחלת יום חדש (הנפוץ)

```
שלום Claude.

אני מתחיל היום [Day X — שם המשימה הראשית, למשל "Day 3 — Morning Agent end-to-end"].

הקבצים שצירפתי:
- 00_START_HERE.md
- 02_PROGRESS_LOG.md
- 03_ROADMAP_DAYS_3_TO_14.md
- [רלוונטי לפי המשימה]

קרא את הקבצים. תאשר שאתה מבין:
1. איפה אנחנו במצב הפרויקט
2. מה היעד של היום
3. אילו pitfalls אתה מצפה לראות

תגיד "מוכן" כשאתה מסיים, ואז אני אכוון אותך לצעד הראשון.

זמן נוכחי אצלי: [HH:MM] — אם אתה חושב שהמשימה גדולה מדי לזמן הזה, תגיד לי לפני שמתחילים.
```

### תרחיש 2 — תיקון bug

```
שלום Claude.

יש לי bug:

**מה ניסיתי:** [תיאור הפעולה]
**מה ציפיתי:** [התוצאה הצפויה]
**מה קרה:** [התוצאה האמיתית]
**שגיאה (אם יש):**
```
[הדבק את ה-error המלא — terminal output, browser console, screenshot text]
```

הקבצים שצירפתי:
- 00_START_HERE.md
- 08_KNOWN_ISSUES.md (אולי יש שם תשובה)
- [קבצים רלוונטיים — schema אם DB issue, code patterns אם code issue]

קרא קודם את ה-Known Issues. אם הבעיה שם — תגיד לי. אם לא — תכוון אותי לדיאגנוסטיקה.
```

```
הקבצים שצירפתי:

```

```
[המשימה הבאה כפי שהוצגה]

הקבצים:
```

### תרחיש 6 — סקירה / planning שבועי

```
שלום Claude.

אני רוצה לעצור ולעשות סקירה לפני שממשיכים.

הקבצים:
- 00_START_HERE.md
- 02_PROGRESS_LOG.md (כל הימים)
- 03_ROADMAP_DAYS_3_TO_14.md

קרא הכל. עכשיו תענה לי על:
1. **מה הושג עד עכשיו** (תקציר 5 שורות)
2. **איפה אנחנו ביחס ל-roadmap המקורי** (לפני / על / אחרי)
3. **מה הסיכונים הגדולים** ל-2 השבועות הבאים
4. **אם הייתי בונה את זה לבד מאפס** — מה היית עושה אחרת?

אני רוצה תשובה כנה, לא diplomatic. אם משהו דחוס מדי או מוקדם מדי — תגיד.
```

## הודעות "סוף יום" (איך לעצור נכון)

לפני שאתה סוגר חלון:

```
Claude, אני עוצר להיום. תעשה לי טובה:

1. **תקציר 5 שורות** של מה שעשינו היום
2. **רשימה של 3-5 דברים שנשארו פתוחים** (TODOs)
3. **commit message** מוצע לכל השינויים שטרם committed
4. **השורה ההכרחית** ל-`02_PROGRESS_LOG.md` שאוסיף

אחרי שאתה עונה — אני סוגר. תודה.
```

זה ייתן לך:
- ✅ Recap מוצק לעדכן את ה-PROGRESS_LOG
- ✅ TODOs לפתיחה ביום הבא
- ✅ Commit message מוכן
- ✅ ידיעה איפה אתה במצב

## אנטי-אנטי-תבנית (מה לא לכתוב)

❌ "המשך מאיפה שעצרנו"
→ Claude לא זוכר. תגיד מה עשינו.

❌ "תעשה לי משהו עם הסוכנים"
→ עמום מדי. תהיה ספציפי.

❌ "תכתוב לי קוד"
→ איזה קוד? לאיזה קובץ? לאיזה מטרה?

❌ "זה לא עובד"
→ מה ניסית? מה ציפית? מה קרה? error?

❌ "סבבה ירכב"
→ אל תאשר עיוור. תקרא את הקוד שאני נותן.

✅ "שגיאה X בקובץ Y שורה Z כשהפעלתי פקודה W. ציפיתי ל-A, קיבלתי B. הקוד הוא: ..."

## ערכת חירום — אם השיחה נשבר ב-context

אם Claude מתחיל לשגות, לחזור על דברים, או לשכוח קונטקסט — סימן ש-context window מתמלא. תפתח שיחה חדשה!

**טקסט להעתיק:**

```
Claude, נראה לי שאיבדנו context. אני פותח שיחה חדשה.

לפני זה — תכתוב לי בכמה שורות:
1. איפה אנחנו במשימה
2. מה הקובץ האחרון שערכנו
3. מה השלב הבא שתכננו
4. אילו קבצים פתוחים / לא saved

אעתיק את זה לשיחה הבאה.
```

ואז העתק את התשובה לתחילת השיחה החדשה (תרחיש 5 — "המשך ישיר").
